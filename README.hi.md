एक्सआरपी लेजर (XRP Ledger) को एक ही बार में सीखें। कोई खाता नहीं। कोई असली पैसा नहीं। सिर्फ आप और लेजर।

एक्सआरपीएल कैंप आपको 6 पाठों के माध्यम से मार्गदर्शन करता है - "लेजर क्या है?" से लेकर एक फंडेड वॉलेट, एक पुष्टि किए गए भुगतान और एक पोर्टेबल प्रमाणपत्र तक - लगभग 10 मिनट में।

## इंस्टॉल करें

```bash
pipx install xrpl-camp
```

या pip के साथ:

```bash
pip install xrpl-camp
```

## क्विक स्टार्ट

```bash
xrpl-camp start
```

यह सभी 6 पाठों के माध्यम से निर्देशित प्रक्रिया चलाता है:

1. **मानसिक मॉडल** — एक्सआरपीएल क्या है (खाते, बैलेंस, लेनदेन, मेमो)
2. **वॉलेट बनाएं** — एक टेस्टनेट कीपेयर (keypair) जेनरेट करें
3. **वॉलेट में फंड डालें** — फ़ॉसेट (faucet) से मुफ्त टेस्ट एक्सआरपी प्राप्त करें
4. **भुगतान भेजें** — लेजर को एक मेमो लिखें (स्वयं भुगतान, 1 ड्रॉप)
5. **लेनदेन सत्यापित करें** — देखें कि आपने क्या लिखा है
6. **प्रमाणपत्र** — आपको जो कुछ भी किया है, उसका एक पोर्टेबल, सत्यापन योग्य रिकॉर्ड प्राप्त करें

## कमांड

| कमांड | यह क्या करता है |
|---------|-------------|
| `xrpl-camp start` | सभी 6 पाठों के माध्यम से निर्देशित प्रक्रिया |
| `xrpl-camp start --dry-run` | नेटवर्क कॉल किए बिना पूरी प्रक्रिया चलाएं |
| `xrpl-camp wallet create` | एक टेस्टनेट वॉलेट बनाएं |
| `xrpl-camp wallet show` | अपने वॉलेट का पता प्रदर्शित करें |
| `xrpl-camp fund` | टेस्टनेट फ़ॉसेट के माध्यम से अपने वॉलेट में फंड डालें |
| `xrpl-camp fund --dry-run` | देखें कि फंडिंग से क्या होगा, बिना नेटवर्क के |
| `xrpl-camp send --memo "hello"` | एक कस्टम मेमो के साथ स्वयं भुगतान भेजें |
| `xrpl-camp send --dry-run` | भुगतान का अनुकरण करें |
| `xrpl-camp verify --tx <hash>` | लेजर पर एक लेनदेन सत्यापित करें |
| `xrpl-camp certificate` | प्रमाणपत्र + प्रूफ पैक जेनरेट करें |
| `xrpl-camp reset` | सभी डेटा हटाएं (इसके लिए टाइप करके पुष्टि की आवश्यकता होती है) |

## आपको अंततः क्या मिलता है

- एक फंडेड टेस्टनेट वॉलेट (`.xrpl-camp/wallet.json` — स्थानीय, gitignored)
- एक्सआरपीएल टेस्टनेट पर एक पुष्टि किया गया भुगतान
- एक सत्यापन रिपोर्ट जो बिल्कुल दिखाती है कि लेजर ने क्या रिकॉर्ड किया
- एक प्रमाणपत्र (`xrpl_camp_certificate.json`) — साझा करने के लिए सुरक्षित, कोई निजी कुंजी नहीं
- एक प्रूफ पैक (`xrpl_camp_proof_pack.json`) — छेड़छाड़-रोधी, SHA-256 हैश

## ड्राई-रन मोड

प्रत्येक नेटवर्क कमांड `--dry-run` का समर्थन करता है। यह दिखाता है कि बिना किसी नेटवर्क कॉल या डेटा बदले क्या होगा। यह आत्मविश्वास बढ़ाने और डिबगिंग के लिए उपयोगी है।

## प्रूफ पैक

प्रूफ पैक आपके द्वारा किए गए हर चीज का एक छेड़छाड़-रोधी रिकॉर्ड है:

- वॉलेट पता और नेटवर्क
- पाठ पूरा होने के समय
- एक्सप्लोरर यूआरएल के साथ लेनदेन आईडी
- टूल संस्करण
- पूरी सामग्री का SHA-256 हैश

कोई भी व्यक्ति हैश को सत्यापित कर सकता है ताकि यह पुष्टि हो सके कि फ़ाइल संपादित नहीं की गई है।

## एंडपॉइंट ओवरराइड

डिफ़ॉल्ट रूप से, एक्सआरपीएल कैंप सार्वजनिक एक्सआरपीएल टेस्टनेट नोड का उपयोग करता है। किसी अन्य एंडपॉइंट का उपयोग करने के लिए:

```bash
export XRPL_CAMP_RPC_URL="https://your-node:51234/"
xrpl-camp fund
```

## सुरक्षा

आपका सीड (निजी कुंजी) स्थानीय रूप से संग्रहीत किया जाता है और इसे कभी भी प्रमाणपत्र या प्रूफ पैक में शामिल नहीं किया जाता है।
यह टूल केवल एक्सआरपीएल **टेस्टनेट** का उपयोग करता है - टेस्ट एक्सआरपी का कोई वास्तविक मूल्य नहीं है।
कोई टेलीमेट्री नहीं, कोई विश्लेषण नहीं, कोई "फोन-होम" नहीं - केवल नेटवर्क कॉल एक्सआरपीएल टेस्टनेट पर जाते हैं।

विस्तार के लिए [SECURITY.md](SECURITY.md) देखें।

## खतरे का मॉडल

| खतरा | निवारण |
|--------|-----------|
| प्रमाणपत्र के माध्यम से सीड का रिसाव | प्रमाणपत्र पीढ़ी में स्पष्ट रूप से सीड को शामिल नहीं किया गया है; `certificate_has_seed()` सुरक्षा जांच |
| प्रूफ पैक के माध्यम से सीड का रिसाव | प्रूफ पैक पीढ़ी में सीड को शामिल नहीं किया गया है; `proof_pack_has_seed()` सुरक्षा जांच |
| गिट में सीड | `.xrpl-camp/` को gitignored किया गया है; `wallet show` कभी भी सीड प्रदर्शित नहीं करता है |
| मेननेट पर टेस्टनेट सीड का पुन: उपयोग | वॉलेट बनाते समय एक चेतावनी प्रदर्शित होती है |
| मेमो सामग्री का एक्सपोजर | सभी मेमो डिफ़ॉल्ट रूप से सार्वजनिक होते हैं; भेजने से पहले उपयोगकर्ताओं को चेतावनी दी जाती है |
| प्रूफ पैक में छेड़छाड़ | SHA-256 अखंडता हैश; `verify_proof_pack()` संशोधन का पता लगाता है |
